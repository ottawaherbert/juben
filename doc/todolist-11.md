# O.M.N.I. Studio 核心架构与提示词重构任务清单 (TODO List 11)

## 目标
构建**“高可控的 AI 编剧副驾 (Co-Pilot) 架构”**。彻底摒弃“AI 强制套模版”的死板逻辑，将系统重构为**“智能推荐 -> 用户干预/修改 -> 迭代生成”**的共创模式，确保创作者对电影、网剧、短剧的全流程拥有绝对的掌控权。

## 任务列表

### 1. 交互范式重构：确立“推荐与共创”机制 (Human-in-the-Loop)
- [ ] **全局引入“生成选项卡 (Generation Options)”**：在 Sequence、Scene、Beat 的任何生成环节，AI 不再直接覆盖结果，而是提供 2-3 个**推荐方案 (Options)** 供用户选择。
- [ ] **支持“自定义指令 (Custom Prompt) 介入”**：在每个生成节点（如拆解场景前），提供一个输入框：“*您对这一段有什么具体想法？（例如：这里我想加一场追车戏，或者，这一集是独立的单元剧故事）*”。AI 必须优先服从用户的自定义指令。
- [ ] **节点级局部重写 (Partial Regenerate)**：允许用户选中某个不满意的 Sequence 或 Scene，输入修改意见，仅针对该局部进行重新生成，而不影响上下文。

### 2. 柔性多体裁数据结构 (Flexible Multi-Format Data Model)
- [ ] **提供体裁“预设模版”，但允许打破**：
  - **🎬 电影 (Movie)**: 默认推荐 [宏观结构点 -> Sequence -> Scene] 结构。
  - **📺 剧集 (TV/Web Series)**: 
    - 提供模式选择：**连续剧 (Serialized)**（强调单集悬念与多线交织） vs **单元剧 (Episodic)**（单集闭环，独立故事）。
    - 默认层级：[季 -> 单集 -> 幕/场景]。
  - **📱 短剧 (Short Drama)**: 默认推荐 [情绪曲线 -> 单集 -> 情绪节拍] 结构。
- [ ] **结构自定义能力**：用户可以随时在 AI 生成的结构上进行增删改。例如，AI 推荐了 8 个 Sequence，用户可以手动删成 6 个，或者手动插入一个特殊的 Sequence，后续的 AI 生成必须基于用户修改后的结构进行。

### 3. 提示词 (Prompt) 策略重构：从“强制指令”到“意图驱动”
- [ ] **废弃所有“硬性数学指标”和“强制剧情要求”**。
- [ ] **Sequence / 单集大纲生成**：
  - **Prompt 逻辑**：“请作为编剧顾问，基于当前的故事背景和用户选择的体裁（如：单元剧），为您**推荐**接下来的叙事段落。如果用户提供了具体想法：[{{用户输入}}]，请**严格按照用户的想法**进行扩展和细化。请提供 2 个不同走向的方案供选择。”
- [ ] **Scene 场景拆解**：
  - **Prompt 逻辑**：“请分析当前段落的上下文。用户希望这一段的重点是：[{{用户输入，如：安静的对话}}]。请顺应这一意图，将段落拆解为具体的场景。不要生搬硬套任何好莱坞模版，确保逻辑自然连贯。”
- [ ] **剧本 (Script) 扩写**：
  - **Prompt 逻辑**：“请根据场景大纲撰写剧本。请参考用户设定的预期时长/节奏。如果用户要求快节奏，请提高动作和台词密度；如果用户要求细腻，请增加微表情和潜台词的描写。一切以用户的创作意图为准。”

### 4. 核心原则贯彻
- [ ] **“创作者本位”审查**：检查前端 UI 和后端 Prompt，确保没有任何一处逻辑是“AI 替用户做决定且不可修改”的。系统只做“提供灵感、完善细节、执行指令”的工作，方向盘必须永远在用户手里。
