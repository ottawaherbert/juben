# O.M.N.I. Studio 系统专业度与 UI/UX 审查报告

作为一名拥有丰富电影行业经验的导演系博士及架构师，我对当前 O.M.N.I. Studio 系统的 UI 界面、制作流程、排版及交互细节进行了深度审查。总体而言，系统架构具备了 AI 辅助影视制作的雏形，但在“专业电影工业标准”和“极致的用户体验”上，仍有较大的提升空间。

以下是详细的审查结果与优化方案：

## 一、 UI/UX 视觉与排版优化 (Typography & Visuals)

1. **字体系统 (Typography) 缺乏电影工业感**
   - **现状**：全局使用了 `Inter` 和 `JetBrains Mono`，过于偏向传统 SaaS 或程序员工具。
   - **改进方案**：
     - **剧本编辑区**：必须强制使用标准的剧本字体（如 `Courier Prime` 或 `Courier New`），字号建议设定为 `12pt` 对应的大小，行距严格遵循好莱坞剧本标准。
     - **标题与展示区**：引入具有电影质感的衬线字体（如 `Playfair Display` 或 `Cinzel`）作为大标题（H1/H2），保留 `Inter` 作为正文阅读字体，提升整体的“艺术气质”。
2. **色彩对比度与层级 (Color Contrast & Hierarchy)**
   - **现状**：大量使用 `neutral-900/950` 搭配 `neutral-400/500`，在某些显示器上对比度过低，长时间工作容易导致视觉疲劳。
   - **改进方案**：提升关键文本的对比度（如将次要文本提升至 `text-neutral-300`）。高亮色 `emerald-500` 很好，但可以引入更多辅助色（如琥珀色代表警告/修改，紫色代表 AI 生成内容）来区分不同的信息层级。
3. **滚动条与操作区 (Scrollbars & Touch Targets)**
   - **现状**：自定义的细滚动条在鼠标操作时不够友好。
   - **改进方案**：在 Hover 状态下适当加宽滚动条，确保拖拽体验。按钮的点击区域（Touch Target）需确保至少 `44x44px`，特别是在移动端或平板设备上。

## 二、 制作流程专业度审查 (Workflow Professionalism)

标准的电影前后期流程为：**概念/大纲 (Pitch/Treatment) -> 剧本 (Script) -> 剧本拆解 (Script Breakdown) -> 分镜 (Storyboard) -> 动态分镜/预演 (Animatic/Previs) -> 拍摄/生成 (Production) -> 剪辑/后期 (Post-Production)**。

1. **剧本拆解 (Script Breakdown) 环节缺失**
   - **现状**：从剧本直接跳到分镜，缺乏专业的拆解过程。
   - **改进方案**：在“剧本”与“分镜”之间增加“剧本拆解”功能。AI 应自动高亮并提取剧本中的角色 (Cast)、道具 (Props)、场景 (Sets)、特效 (VFX)、音效 (SFX) 等元素，生成标准的拆解表（Breakdown Sheet）。
2. **分镜 (Storyboard) 缺乏专业摄影机语言**
   - **现状**：分镜提示词较为基础。
   - **改进方案**：分镜面板必须包含专业的摄影机参数选项：景别（CU, MCU, WS, VWS）、机位（High Angle, Low Angle, Dutch Tilt）、运动（Pan, Tilt, Dolly, Tracking）、焦段（14mm, 50mm, 200mm）。AI 生成时需严格遵循这些参数。
3. **摄影棚 (Studio) 缺乏非线性剪辑 (NLE) 逻辑**
   - **现状**：Studio 页面左侧是分镜列表，中间是预览，右侧是配音。更像是一个播放器而非工作台。
   - **改进方案**：引入真正的多轨道时间线（Timeline）UI。包含视频轨、对白轨、音效轨、配乐轨。支持拖拽调整片段长度（Trim）、切割（Split）和转场（Transitions）。

## 三、 核心功能细节打磨 (Core Feature Polish)

1. **剧本编辑器 (Script Editor) 的交互**
   - **现状**：支持拖拽和 AI 局部重写，但纯文本模式和块模式的切换不够无缝。
   - **改进方案**：实现类似 Final Draft 或 Highland 2 的无缝输入体验。通过快捷键（如 Tab 切换角色和对白，Enter 切换动作和场景标题）自动格式化文本，而不是完全依赖 UI 按钮。
2. **灵感注入端 (Home) 的选项呈现**
   - **现状**：国家/地区和导演/演员的选项是纯文本按钮，略显枯燥。
   - **改进方案**：使用带有代表性电影剧照或情绪板（Moodboard）的卡片式 UI 来展示不同的视觉风格，让导演在选择时有更直观的视觉参考。

## 四、 发现的 Bug 与不合理之处 (Identified Bugs & Issues)

1. **Home.tsx 渲染逻辑隐患**：在 `styleMode === 'region'` 和 `director-actor` 的切换中，虽然逻辑闭合，但状态重置不彻底。切换模式时应清空另一模式的输入值，避免 AI 提示词拼接混乱。
2. **Studio.tsx 状态同步延迟**：在输入配音台词时，使用了 500ms 的防抖（debounce）来更新 Store。但在快速切换分镜时，可能会导致上一个分镜的台词被错误地保存到当前分镜中。需要清理 useEffect 的 timeout 或在切换分镜时强制 flush。
3. **Projects.tsx 布局溢出**：在小屏幕下，项目卡片的标题如果过长，虽然有 `truncate`，但底部的标签栏（如“电影长片”和“Chapters”）可能会因为空间挤压而换行错乱，建议使用 `flex-wrap` 或进一步优化卡片内部的 Flex 布局。
4. **导出格式支持不足**：目前支持 `.fountain` 导出，但工业界仍大量使用 PDF 和 `.fdx` (Final Draft)。建议增加将 Fountain 转换为标准排版 PDF 的导出功能。
