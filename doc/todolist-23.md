# O.M.N.I. Studio 分镜与视觉提示词重构方案 (todolist-23)

## 1. 核心问题诊断 (Problem Definition)
当前系统在生成分镜时，采用的是“单次调用、全量输出”的重型提示词（Prompt Over-engineering）。大模型需要同时处理镜头拆分、物理净化、资产解耦、风格蒸馏等复杂指令，导致：
*   **叙事逻辑崩塌**：模型注意力被规则分散，无法专注“导戏”，导致分镜缺乏逻辑。
*   **视觉污染与复读机**：模型为了满足格式要求，机械地将全局风格（如“汉江背景”）强塞入每一个镜头，导致生图失败。
*   **黑盒操作**：用户无法在分镜生成后、提示词生成前进行干预和修改。

## 2. 重构方案：二段式解耦生产流 (Two-Stage Decoupled Workflow)
将原有的“一键出图”拆分为“导演脑”与“摄影脑”两个独立的、用户可干预的流程。**注意：根据本地化需求，所有生成的提示词必须为纯中文。**

### 第一阶段：导演脑 (Director's Brain) —— 生成专业分镜脚本
*   **目标**：只管“戏”和“调度”，不考虑具体的画质词。将剧本拆解为纯物理的、符合导演风格的镜头清单。
*   **输入**：剧本内容 + 创作视点（仅作为节奏和景别偏好的参考）。
*   **输出结构**：
    ```json
    {
      "shotNumber": 1,
      "shotSize": "MCU", // 景别
      "cameraAngle": "Low", // 角度
      "cameraMovement": "Push-in", // 运镜
      "visualAction": "主角的手指在冰冷的手机屏幕上剧烈颤抖...", // 纯物理动作描述
      "lightingAtmo": "冷色调，高反差侧光", // 光影意图
      "duration": 5, // 时长
      "motivation": "通过低角度近景增强压迫感..." // 导演动机
    }
    ```
*   **交互**：用户点击“生成分镜”，系统展示分镜列表。用户可以手动编辑、调整每一个镜头的参数和动作描述。

### 第二阶段：摄影脑 (Cinematographer's Brain) —— 生成视觉提示词
*   **目标**：拿到导演的分镜脚本，结合全局的风格滤镜，翻译成生图模型能理解的**中文**视觉标签。
*   **输入**：用户确认后的单条或多条分镜数据 + 创作视点（作为色彩、质感滤镜）。
*   **输出结构**：
    ```json
    {
      "shotNumber": 1,
      "imagePrompt": "中近景，仰拍，男人的手在破裂的手机屏幕上颤抖，玻璃倒映出血丝的眼睛，冷蓝色调，强烈的侧面打光，高对比度，电影级质感，8k分辨率", // 纯中文提示词
      "videoPrompt": "镜头缓慢推近颤抖的手和反光的屏幕", // 中文视频动态描述
      "visualSummary": "中近景仰拍，特写颤抖的手指与手机屏幕上的倒影，冷调高反差光影。" // 中文摘要
    }
    ```
*   **交互**：用户在确认分镜后，点击“生成提示词”。系统生成后，用户可以固化、查看并手动编辑这些提示词，然后再进行图片/视频的生成。

## 3. 实施计划 (Action Items)

- [x] **Phase 1: 数据结构重构** - 修改 `store/useProjectStore.ts` 中的 `Shot` 接口，分离分镜基础数据与视觉提示词数据。
- [x] **Phase 2: 导演脑 Prompt 设计与集成** - 重写 `promptBuilder.ts` 中的分镜生成逻辑，移除所有关于生图提示词的要求，专注物理动作和镜头调度。
- [x] **Phase 3: 摄影脑 Prompt 设计与集成** - 新增视觉提示词生成逻辑，接收分镜数据和创作视点，严格要求输出**纯中文**的 `imagePrompt` 和 `videoPrompt`。
- [x] **Phase 4: UI/UX 流程改造** - 改造 `Storyboard.tsx` 界面：
    - 将原来的“一键生成”拆分为“生成分镜”和“生成提示词”两步操作。
    - 提供分镜参数（景别、角度、动作等）的编辑界面。
    - 提供提示词（imagePrompt, videoPrompt）的查看和编辑界面。
- [x] **Phase 5: 资产与参数自动组装** - 在最终请求生图 API 时，由代码在后台自动拼接 `--ar`、`--cref` 等控制参数，保持前端提示词的纯净。
