# O.M.N.I. 系统优化与重构计划 (To-Do List 4)

## 核心问题诊断与分析

当前系统确实存在“半成品”的体验感，主要原因在于**各个模块之间的数据孤岛**以及**缺乏自动化的工作流串联**。用户需要手动在不同页面之间同步信息，这违背了 AI 辅助创作“流畅、连贯”的初衷。

以下是对您提出的几个核心问题的深度剖析：

### 1. AI Provider 切换失败问题
**问题原因**：在之前的代码中，`import.meta.env.VITE_AI_PROVIDER` 可能被底层运行环境默认注入了 `"google"`，导致 `|| "deepseek"` 的后备逻辑失效，从而一直调用 Gemini。
**解决方案**：已在 `src/config/ai.ts` 中强制将 `provider` 设置为 `"deepseek"`。现在文本生成将严格走 DeepSeek 接口（注：由于 DeepSeek 不支持生图，生图接口 `generateAIImage` 仍保留使用 Gemini）。

### 2. 角色、场景、道具、配音与资产库的联动关系
**现状**：目前这些元素散落在“项目圣经”、“摄影棚”和“资产库”中，彼此割裂。资产库更像是一个单纯的文件上传区，而不是项目的“视觉/听觉基石”。
**理想逻辑**：
- **项目圣经 (Bible)** 是**定义层**：用文字描述角色是谁、场景长什么样。
- **资产库 (Assets)** 是**具象层**：是圣经的视觉和听觉体现。
- **联动机制**：
  1. 圣经中一旦确立了角色/场景/道具，它们应该**自动同步**到资产库中，形成一个个“待生成”的资产卡片。
  2. 用户在资产库中点击“生成参考图”，AI 根据圣经中的描述生成一致性参考图（Reference Image）。
  3. 这些参考图在后续的“分镜生成”中，作为垫图或 Prompt 约束，确保画面一致性。

### 3. 项目圣经中角色缺失与生成时机
**现状**：圣经中的角色通常只在项目初期（一句话大纲阶段）生成一次。如果后续剧本中出现了新角色，系统不会自动更新圣经。
**理想逻辑**：
- **动态提取 (Entity Extraction)**：在“剧本生成”或“分镜生成”环节，系统应在后台静默运行一个 AI 任务，分析当前文本：“这里是否出现了圣经中没有的新角色/场景/道具？”
- 如果有，系统应自动将其补充到项目圣经中，并同步推送到资产库等待生成参考图。

### 4. 摄影棚配音与角色的自动关联
**现状**：在摄影棚生成配音时，用户需要手动选择文本，再手动选择配音员，效率极低。
**理想逻辑**：
- 在项目圣经或资产库中，每个角色都应该有一个绑定的“默认音色 (Voice Profile)”。
- 当剧本生成完毕后，系统应自动解析出每一句台词属于哪个角色。
- 在摄影棚中，系统直接列出所有台词，并**自动选中**该角色绑定的音色，用户只需点击“一键生成全部配音”即可。

## 专家级系统梳理与待办清单 (To-Do List 4)

为了让 O.M.N.I. 达到“流程通畅、好用、专业”的目标，我们需要进行以下深度的架构和交互重构：

### 第一阶段：打通数据血脉 (数据模型重构) - ✅ 已完成
- [x] **重构全局状态 (Store)**：确保 Bible（圣经）、Assets（资产）、Script（剧本）共享同一个数据上下文。
- [x] **建立实体 ID 关联**：给每个角色、场景、道具分配唯一 ID。剧本中的台词必须关联到角色 ID；分镜中的画面必须关联到场景 ID。
- [x] **资产库自动化**：监听 Bible 数据的变化。当 Bible 中新增角色/场景时，自动在 Assets 中创建对应的占位符（Placeholder）。

### 第二阶段：资产库与一致性工作流 - ✅ 已完成
- [x] **资产库“一键生成”功能**：在资产库的占位符卡片上增加“生成参考图”按钮，自动读取 Bible 中的描述作为 Prompt 调用生图接口。
- [x] **角色音色绑定**：在角色资产卡片中，增加“选择/试听音色”功能，将特定 TTS 音色与角色 ID 永久绑定。
- [x] **生图一致性注入**：修改分镜生图的 Prompt 逻辑，在生成分镜图片时，自动将该场景/角色的“参考图描述”或“参考图特征”注入到 Prompt 中。

### 第三阶段：动态实体提取 (解决角色缺失) - ✅ 已完成
- [x] **剧本后处理钩子 (Post-Script Hook)**：每次生成或修改剧本后，触发一个后台 AI 任务，提取剧本中的所有实体。
- [x] **实体对比与合并**：将提取出的实体与当前 Bible 进行对比，发现新角色/场景/道具时，提示用户“发现新实体，是否添加到项目圣经与资产库？”。

### 第四阶段：摄影棚与剧本的深度整合 - ✅ 已完成
- [x] **剧本台词结构化**：目前的剧本可能是纯文本，需要将其结构化为 `[{ characterId, text, emotion }]` 的格式。
- [x] **摄影棚自动化**：摄影棚页面不再是空白的生成器，而是直接读取结构化剧本，按时间线展示所有台词，并自动匹配已绑定的角色音色。
- [x] **批量生成与导出**：支持一键生成当前集数的所有配音，并按顺序排列，方便后续合成。

### 第五阶段：交互与易用性优化 (细节打磨) - ✅ 已完成
- [x] **全局进度与状态指示**：在顶部导航栏增加“项目健康度”或“待办事项”提示（例如：“您有 3 个新角色尚未生成参考图”）。
- [x] **错误边界与重试机制**：AI 接口调用失败时，提供优雅的错误提示和一键重试按钮，而不是让页面卡死。
- [x] **DeepSeek 深度适配**：针对 DeepSeek 的特性（如更强的逻辑推理能力），优化剧本和分镜的 Prompt 结构，强制要求返回严格的 JSON 格式以配合前端结构化渲染。
