# O.M.N.I. 系统透明化与上下文连贯性重构 (To-Do List 8)

基于您的反馈，当前系统存在两个核心痛点：
1. **上下文割裂 (Context Fragmentation)**：AI 生成内容（特别是节拍大纲和剧本）时，缺乏对全局设定、前序剧情的有效参考，导致连续剧/电影的剧情不连贯，而单元剧又缺乏灵活的隔离机制。
2. **AI 黑盒效应 (Blackbox AI)**：所有的 Prompt（提示词）都是硬编码在系统底层的，用户无法查看、修改和保存，导致系统难以调优，缺乏掌控感。

为了彻底解决这些问题，将系统从“黑盒”变成“白盒”，我们制定了以下重构计划：

## 第一阶段：提示词透明化与模板引擎 (Prompt Transparency & Engine) - 优先级：最高

- [x] **Task 1.1: 建立全局提示词模板系统 (Prompt Template Store)**
  - 将目前散落在各个页面（Home, Episodes, Structure, Script, Storyboard）的硬编码 Prompt 提取出来。
  - 引入简单的模板变量语法（如 `{{logline}}`, `{{episode_title}}`, `{{previous_summary}}`），使 Prompt 结构化。
- [x] **Task 1.2: 开发“AI 提示词拦截与编辑弹窗” (Prompt Editor Modal)**
  - 在系统中任何触发 AI 生成的地方（点击“魔法棒”按钮），不再直接发送请求，而是弹出此窗口。
  - **功能**：
    1. 展示当前即将发送给 AI 的**完整提示词**（变量已替换为实际内容）。
    2. 允许用户在文本框中**直接编辑**和微调提示词。
    3. 提供**“确认并发送”**按钮。
- [x] **Task 1.3: 提示词的“固化保存”机制**
  - 在弹窗中增加“保存为默认模板”功能。用户修改后的 Prompt 结构可以保存到项目设置或全局设置中，后续生成同类内容时默认使用用户调优过的模板。

## 第二阶段：动态上下文注入系统 (Dynamic Context Injection) - 优先级：最高

- [x] **Task 2.1: 引入“上下文选择器” (Context Selector)**
  - 在 Prompt Editor Modal 中，增加一个可视化的“参考上下文”侧边栏/折叠面板。
  - 用户可以勾选本次生成需要让 AI 知道的信息，例如：
    - [x] 全局 Logline 与核心冲突
    - [x] 特定角色档案（可多选）
    - [x] 上一集/上一幕的剧情梗概
    - [x] 整个篇章 (Arc) 的宏观目标
- [x] **Task 2.2: 基于项目类型的默认上下文策略**
  - **电影 (Movie) / 连续剧 (TV Series)**：默认勾选“上一集/上一场戏梗概”和“全局核心设定”，确保剧情连贯、伏笔不断。
  - **单元剧 (Anthology) / 微短剧 (Short Drama)**：默认仅勾选“全局核心设定”和“当前集大纲”，确保单集独立性，避免 AI 混淆其他集数的内容。

## 第三阶段：各核心模块全面接入新工作流 (Integration) - 优先级：高

- [x] **Task 3.1: 改造“分集 (Episodes)”与“结构 (Structure)”模块**
  - 重点解决您提到的“节拍大纲生成割裂”问题。在生成节拍大纲时，弹窗将明确展示 AI 参考了哪些前置剧情，并允许您手动补充“本集需要承接的上一集悬念”。
- [x] **Task 3.2: 改造“剧本 (Script)”与“分镜 (Storyboard)”模块**
  - 剧本重写、台词润色、画面 Prompt 生成等所有 AI 动作，全面接入“拦截弹窗”，让您清晰看到 AI 是如何被指导的，并随时修正导演意图（如强制要求某种镜头语言）。
